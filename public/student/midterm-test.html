Computational Thinking

เปลี่ยนวิธีคิดให้เป็นระบบด้วย 4 องค์ประกอบสำคัญ พร้อมเจาะลึกโครงสร้างอัลกอริทึม พื้นฐานสำคัญสู่การเป็นโปรแกรมเมอร์มืออาชีพ

เริ่มบทเรียน ทำแบบทดสอบเลย

1. แนวคิดเชิงคำนวณ (Computational Thinking)

กระบวนการแก้ปัญหาแบบนักคอมพิวเตอร์ ไม่ใช่แค่การเขียนโค้ด แต่คือ "วิธีคิด" ที่นำไปใช้ได้กับทุกเรื่อง

Decomposition

การแตกปัญหาใหญ่เป็นปัญหาย่อย

การแยกส่วนประกอบของปัญหาที่ซับซ้อนออกเป็นส่วนเล็กๆ เพื่อให้จัดการได้ง่ายขึ้น

ตัวอย่าง:

ชีวิตจริง: การสร้างบ้าน แบ่งเป็น งานฐานราก, งานโครงสร้าง, งานระบบไฟ, งานตกแต่ง
Programming: แบ่งโปรแกรมใหญ่เป็นฟังก์ชัน (Functions) ย่อยๆ

Pattern Recognition

การจดจำรูปแบบ

การหารูปแบบหรือความคล้ายคลึงกันของปัญหา เพื่อนำวิธีการเดิมมาใช้ซ้ำ (Reusability)

ตัวอย่าง:

ชีวิตจริง: การขับรถ รถต่างยี่ห้อมีพวงมาลัยและเบรกเหมือนกัน เรียนรู้ครั้งเดียวขับได้หมด
Programming: การใช้ Loop วนซ้ำกับข้อมูลที่มีโครงสร้างเหมือนกัน

Abstraction

ความคิดเชิงนามธรรม

การมุ่งเน้นเฉพาะสาระสำคัญ (Main Idea) และตัดรายละเอียดปลีกย่อยที่ไม่จำเป็นทิ้งไป

ตัวอย่าง:

ชีวิตจริง: แผนที่รถไฟฟ้า บอกแค่สถานีและเส้นทาง ไม่บอกต้นไม้หรือร้านค้าข้างทาง
Programming: การสร้าง UI ปุ่มกด ผู้ใช้เห็นแค่ปุ่ม ไม่ต้องรู้โค้ดข้างใน

Algorithm Design

การออกแบบอัลกอริทึม

การลำดับขั้นตอนการแก้ปัญหาอย่างเป็นระบบ ตั้งแต่ต้นจนจบ (Step-by-step)

ตัวอย่าง:

ชีวิตจริง: คู่มือประกอบตู้ IKEA หรือสูตรอาหาร
Programming: ขั้นตอนการเรียงลำดับข้อมูล (Sorting) จากน้อยไปมาก

2. อัลกอริทึม (Algorithm)

คุณสมบัติที่ดี

  • 1 ชัดเจน (Unambiguous): ขั้นตอนต้องไม่กำกวม ตีความได้ทางเดียว
  • 2 มีจุดจบ (Finite): ทำงานแล้วต้องจบ ไม่วนลูปไม่รู้จบ
  • 3 มีผลลัพธ์ (Output): ต้องได้ผลลัพธ์ตามที่ต้องการอย่างน้อย 1 อย่าง

เลือกรูปแบบการแสดงผล:

Project: ต้มบะหมี่กึ่งสำเร็จรูป.algo

ภาษาธรรมชาติ (Narrative)

การบรรยายขั้นตอนด้วยภาษาพูดหรือเขียน เหมาะสำหรับสื่อสารกับคนทั่วไป

เข้าใจง่าย อาจตีความผิดได้

ขั้นตอนการทำ:

  1. เริ่มต้นการทำงาน
  2. ต้มน้ำในหม้อให้เดือด
  3. นำก้อนบะหมี่ใส่ชาม แล้วเทน้ำร้อนใส่ให้ท่วม
  4. ปิดฝาชามทิ้งไว้ 3 นาที
  5. เปิดฝาตรวจสอบเส้น
    • เงื่อนไข: ถ้าเส้นยังแข็ง (ไม่สุก) ให้ปิดฝารอต่ออีก 1 นาที แล้วตรวจสอบใหม่
    • เงื่อนไข: ถ้าเส้นนิ่มแล้ว (สุก) ให้ทำข้อต่อไป
  6. ใส่เครื่องปรุงและคนให้เข้ากัน
  7. รับประทาน
  8. จบการทำงาน

รหัสลำลอง (Pseudo Code)

การเขียนจำลองโครงสร้างภาษาโปรแกรม ตัดเรื่องไวยากรณ์ (Syntax) ที่ซับซ้อนออก เพื่อเน้น Logic

pseudo_code_v1.0
BEGIN Cooking_Noodles

    // 1. ขั้นตอนเตรียมการ (Sequence)
    Boil water
    Place noodles in bowl
    Pour hot water
    Wait 3 minutes
    
    // 2. การวนซ้ำเพื่อตรวจสอบ (Loop)
    WHILE (noodles is NOT soft) DO
        Wait 1 minute
        Check noodles status
    ENDWHILE
    
    // 3. ปรุงรสและจบงาน
    Add seasoning
    Stir well
    Eat

END Cooking_Noodles
Keywords: คำสั่งหลักมักใช้ตัวพิมพ์ใหญ่ (BEGIN, WHILE, IF)
Indentation: การย่อหน้าสำคัญมาก เพื่อบอกขอบเขตของคำสั่ง

ผังงาน (Flowchart)

การใช้สัญลักษณ์ภาพมาตรฐาน เพื่อแสดงลำดับการทำงาน เห็นภาพรวมและการไหลของข้อมูลได้ชัดเจนที่สุด

3. โครงสร้างควบคุม (Control Structures)

3 รูปแบบพื้นฐานที่ประกอบกันเป็นโปรแกรมคอมพิวเตอร์ทุกชนิดในโลก

1. การทำงานแบบลำดับ
(Sequence)

ทำทีละคำสั่งจากบนลงล่าง เรียงต่อกันไปเรื่อยๆ ห้ามข้ามขั้นตอน

2. การเลือกทำ/เงื่อนไข
(Selection)

ตัดสินใจเลือกทำทางใดทางหนึ่งตามเงื่อนไข (True/False หรือ Yes/No)

3. การทำงานวนซ้ำ
(Repetition/Loop)

ทำคำสั่งเดิมซ้ำๆ จนกว่าเงื่อนไขจะเปลี่ยน หรือครบจำนวนรอบที่กำหนด

4. สัญลักษณ์ผังงาน (Flowchart Symbols)

มาตรฐาน ANSI/ISO ที่ใช้กันทั่วโลก

Terminator (เริ่มต้น/สิ้นสุด)

จุดเริ่มต้นและจุดสิ้นสุดของการทำงาน

ข้อกำหนด: มีจุด Start ได้แค่จุดเดียว แต่มี Stop ได้หลายจุด

Process (การปฏิบัติงาน)

การคำนวณ หรือกำหนดค่า

ตัวอย่าง: Area = Width * Height, X = X + 1

General I/O (รับ/แสดงผล)

รับหรือแสดงข้อมูลแบบไม่ระบุอุปกรณ์

ตัวอย่าง: Read Score, Print "Hello World"

Decision (การตัดสินใจ)

ตรวจสอบเงื่อนไข ทางออก 2 ทาง (True/False)

ตัวอย่าง: Score >= 50?, Is Member?

Connector (จุดเชื่อมต่อ)

เชื่อมผังงานในหน้าเดียวกัน

วิธีใช้: ใช้อักษร A, B, C เป็นคู่เพื่อระบุจุดเชื่อม

Flow Line (ทิศทาง)

แสดงลำดับขั้นตอนการทำงาน

ข้อควรระวัง: หัวลูกศรต้องชี้ไปยังขั้นตอนถัดไปเสมอ ห้ามปล่อยลอย

ทดสอบความเข้าใจ

ลองตอบคำถาม 5 ข้อ เพื่อวัดระดับความรู้ของคุณ

ข้อที่ 1

หากคุณต้องการเขียนโปรแกรม "ตัดเกรดนักเรียน" โดยพิจารณาแค่คะแนนสอบ ไม่สนใจชื่อหรือเลขที่นักเรียน นี่คือแนวคิดใด?

ข้อที่ 2

สัญลักษณ์ "สี่เหลี่ยมข้าวหลามตัด" (Decision) ต้องมีเส้นลูกศรออกจากรูปกี่เส้นเสมอ?

ข้อที่ 3

การเขียน Pseudo Code (รหัสลำลอง) ข้อใด **ไม่จำเป็น** ต้องคำนึงถึง?

ข้อที่ 4

ข้อที่ 5
