เริ่มบทเรียน

แนวคิดเชิงคำนวณ (Computational Thinking)

4 องค์ประกอบสำคัญในการคิดแก้ปัญหาแบบนักวิทยาศาสตร์คอมพิวเตอร์

Decomposition

การย่อยปัญหา

การแตกปัญหาใหญ่ที่ซับซ้อนให้เป็นปัญหาย่อยๆ เพื่อให้จัดการได้ง่ายขึ้น

ตัวอย่าง:

การจัดงานกีฬาสี แบ่งงานเป็น ฝ่ายสถานที่, ฝ่ายการแสดง, ฝ่ายกีฬา และฝ่ายสวัสดิการ

Pattern Recognition

การจดจำรูปแบบ

การหารูปแบบหรือความคล้ายคลึงของปัญหา เพื่อนำวิธีการเดิมมาประยุกต์ใช้

ตัวอย่าง:

รถยนต์ทุกคันมีล้อ พวงมาลัย และเบรกเหมือนกัน แม้จะต่างยี่ห้อ ขับได้ด้วยวิธีการเดียวกัน

Abstraction

ความคิดด้านนามธรรม

การมุ่งเน้นเฉพาะข้อมูลสำคัญ และตัดรายละเอียดที่ไม่จำเป็นทิ้งไป (Focus on key details)

ตัวอย่าง:

แผนที่รถไฟฟ้า แสดงเฉพาะเส้นทางและสถานี ไม่แสดงต้นไม้ ตึก หรือร้านค้าข้างทาง

Algorithm Design

การออกแบบอัลกอริทึม

การวางลำดับขั้นตอนการแก้ปัญหาอย่างเป็นระบบ ตั้งแต่ต้นจนจบเพื่อให้คนหรือคอมพิวเตอร์ทำตามได้

ตัวอย่าง:

สูตรทำอาหาร ที่บอกวัตถุดิบและขั้นตอนการปรุงทีละข้อ จนได้อาหารสำเร็จ

Step-by-Step

Algorithm (อัลกอริทึม)

อัลกอริทึม คือ กระบวนการแก้ปัญหาที่มีลำดับขั้นตอนชัดเจน ไม่กำกวม โดยมี 3 รูปแบบหลักในการนำเสนอ:

Example: ต้มบะหมี่กึ่งสำเร็จรูป

ภาษาธรรมชาติ (Narrative)

เหมาะสำหรับสื่อสารทั่วไป แต่อาจมีความกำกวม

คำอธิบาย:

เป็นการใช้ภาษาพูดหรือเขียนในการบรรยายขั้นตอนอย่างละเอียด ข้อดีคือง่ายต่อการเข้าใจสำหรับคนทั่วไป แต่ข้อเสียคือคอมพิวเตอร์ไม่สามารถเข้าใจได้โดยตรง และภาษาอาจดิ้นได้

ขั้นตอนการต้มบะหมี่:

  1. เริ่มต้นการทำงาน
  2. เตรียมอุปกรณ์และนำน้ำสะอาดใส่หม้อ ต้มน้ำให้เดือด
  3. แกะซองบะหมี่ นำก้อนบะหมี่ใส่ลงในชาม
  4. เมื่อน้ำเดือดแล้ว เทน้ำร้อนใส่ชามที่มีบะหมี่ให้ท่วมเส้น
  5. นำจานมาปิดฝาชามไว้ รอประมาณ 3 นาที
  6. เปิดฝาและใช้ส้อมตรวจสอบเส้น
    • เงื่อนไข: ถ้าเส้นยังแข็งอยู่ ให้ปิดฝารอต่ออีก 1 นาที
    • ถ้าเส้นนิ่มสุกพอดี ให้ทำขั้นตอนต่อไป
  7. ฉีกซองเครื่องปรุง ใส่ลงในชามและคนให้เข้ากัน
  8. รับประทานบะหมี่
  9. จบการทำงาน

รหัสลำลอง (Pseudo Code)

โครงสร้างคล้ายโค้ด แต่ไม่ใช่ภาษาคอมพิวเตอร์จริง

ช่วยให้นักเขียนโปรแกรมวางแผนโครงสร้างตรรกะ (Logic) ได้ชัดเจนก่อนเริ่มเขียนโค้ดจริง ตัดความกังวลเรื่อง Syntax ภาษาทิ้งไป

BEGIN Cooking_Noodles
    Boil water
    Put noodles into bowl
    Pour hot water into bowl
    Wait 3 minutes
    
    // วนลูปตรวจสอบความสุกของเส้น
    WHILE (noodles is NOT soft) DO
        Wait 1 minute
        Check noodles status
    ENDWHILE
    
    Add seasoning
    Stir well
    Eat
END Cooking_Noodles

ผังงาน (Flowchart)

แสดงความสัมพันธ์ของขั้นตอนด้วยรูปภาพ

ใช้สัญลักษณ์มาตรฐานสากล ทำให้คนต่างภาษาเข้าใจตรงกันได้ง่าย เหมาะสำหรับการตรวจสอบความผิดพลาดของกระบวนการ (Process Flow)

  • สี่เหลี่ยมมน: เริ่มต้น/สิ้นสุด
  • สี่เหลี่ยมผืนผ้า: การกระทำ
  • สี่เหลี่ยมข้าวหลามตัด: การตัดสินใจ

สัญลักษณ์ผังงาน (Flowchart Symbols)

Terminator (จุดเริ่มต้น/สิ้นสุด)

ต้องมีเสมอ! ใช้แสดงจุดเริ่มต้นของการทำงาน และจุดสิ้นสุดเมื่อจบงาน

Tip: ควรมีจุด Start เพียงจุดเดียวใน 1 ผังงาน

Process (การปฏิบัติงาน)

ใช้เมื่อมีการคำนวณ การกำหนดค่าตัวแปร หรือการประมวลผลข้อมูล

Tip: เขียนข้อความสั้นๆ กระชับ เช่น "X = X + 1" หรือ "คำนวณภาษี"

Input / Output (รับ/แสดงผล)

ใช้เมื่อรับค่าเข้ามา (เช่น พิมพ์คีย์บอร์ด) หรือแสดงผลลัพธ์ (เช่น โชว์บนจอ)

Tip: ใช้รูปทรงนี้เมื่อไม่ได้ระบุอุปกรณ์เฉพาะเจาะจง

Decision (การตัดสินใจ)

จุดตรวจสอบเงื่อนไข จะมีลูกศรเข้า 1 เส้น และลูกศรออก 2 เส้นเสมอ (จริง/เท็จ)

Tip: เขียนคำถามในรูปทรง เช่น "อายุ > 18?" หรือ "จริงหรือไม่"

Connector (จุดเชื่อมต่อ)

ใช้เมื่อผังงานยาวเกินไปและต้องตัดไปขึ้นคอลัมน์ใหม่ในหน้าเดียวกัน

Tip: ใช้อักษรภาษาอังกฤษ (A, B, C) ระบุคู่จุดเชื่อม

Flow Line (ทิศทาง)

เส้นลูกศรแสดงลำดับขั้นตอนการทำงานจากบนลงล่าง หรือซ้ายไปขวา

Tip: หัวลูกศรต้องชี้ไปยังขั้นตอนถัดไปให้ชัดเจน

ทดสอบความเข้าใจ

1. การวางแผนจัดงานโรงเรียน โดยแบ่งหน้าที่ให้แต่ละฝ่ายรับผิดชอบ ตรงกับแนวคิดใด?

2. ถ้าต้องการตรวจสอบว่า "คะแนนสอบผ่านเกณฑ์หรือไม่" ควรใช้สัญลักษณ์ใดในผังงาน?

3. ข้อใดคือข้อดีของ "รหัสลำลอง (Pseudo Code)"?
