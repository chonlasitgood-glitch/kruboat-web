01

การออกแบบอัลกอริทึม

Algorithm & Abstraction

แนวคิดเชิงนามธรรม (Abstraction)

คือองค์ประกอบหนึ่งของแนวคิดเชิงคำนวณ (Computational Thinking) หมายถึงการมุ่งเน้นความสำคัญของปัญหาโดย "คัดเลือกเฉพาะข้อมูลที่จำเป็น และตัดรายละเอียดที่ไม่จำเป็นออกไป"

ตัวอย่าง: แผนที่รถไฟฟ้า

ข้อมูลที่จำเป็น (Abstract)
  • ชื่อสถานี
  • จุดเชื่อมต่อสถานี
  • ลำดับก่อน-หลัง
สิ่งที่ไม่จำเป็น (Details)
  • ระยะทางจริงที่คดเคี้ยว
  • ต้นไม้ข้างทาง
  • ความลึกของอุโมงค์

อัลกอริทึม (Algorithm)

คือระเบียบวิธีหรือขั้นตอนวิธีในการแก้ปัญหาที่มีความชัดเจน เรียงลำดับตั้งแต่ต้นจนจบ (Step-by-step) เพื่อให้ได้ผลลัพธ์ที่ถูกต้อง

คุณสมบัติที่ดีของอัลกอริทึม

  • มีความชัดเจน ไม่คลุมเครือ
  • มีลำดับขั้นตอนแน่นอน (1, 2, 3...)
  • มีจุดเริ่มต้นและจุดสิ้นสุดเสมอ
  • ประมวลผลแล้วได้ผลลัพธ์ที่ถูกต้อง

3 รูปแบบการเขียนอัลกอริทึม

1. ภาษาธรรมชาติ

การบรรยายด้วยภาษาพูด/เขียน

"เริ่มจากต้มน้ำให้เดือดจัดก่อน จากนั้นจึงแกะซองใส่บะหมี่กึ่งสำเร็จรูปลงไปต้ม รอประมาณ 3 นาทีจนเส้นนิ่ม แล้วจึงปิดไฟพร้อมเสิร์ฟ"

2. รหัสลำลอง (Pseudocode)

จำลองโค้ดด้วยคำสั่งสั้นๆ

START
  Boil Water
  IF Water_Boiled THEN
    Add Noodle
    Wait 3 mins
  ELSE
    Wait until boiled
END

3. ผังงาน (Flowchart)

ใช้สัญลักษณ์รูปภาพที่เป็นมาตรฐาน

Start
เริ่มต้น/สิ้นสุด
Process
ประมวลผล
I/O
รับ/แสดงผล
ตัดสินใจ
02

การเขียนโปรแกรมเบื้องต้น

Basic Programming

รู้จักกับ "ตัวแปร" (Variable)

ในการเขียนโปรแกรม เราต้องมีที่สำหรับเก็บข้อมูลเพื่อนำไปใช้ต่อ เราเรียกสิ่งนี้ว่า "ตัวแปร"

เปรียบเทียบ: ตัวแปร = กล่อง

  • ชื่อตัวแปร: ป้ายชื่อที่ติดหน้ากล่อง (เช่น Score, Name)
  • ค่าข้อมูล: ของที่ใส่ในกล่อง (เช่น 100, "Somchai")
  • ชนิดข้อมูล: ประเภทของกล่อง (กล่องใส่ตัวเลข, กล่องใส่ข้อความ)

แนะนำ Scratch

โปรแกรมภาษาแบบบล็อก (Block-based) ที่เหมาะสำหรับผู้เริ่มต้น

Motion
เคลื่อนที่ หมุน ไปที่
Looks
พูด เปลี่ยนชุด เปลี่ยนฉาก
Events
เมื่อคลิกธงเขียว, เมื่อกดปุ่ม
Control
วนซ้ำ (Loop), ถ้า...แล้ว (If)

ตัวอย่าง: จากผังงานสู่โปรแกรม

1. ผังงาน (Flowchart)
Start
Show "Hello"
End
2. Python Code
print("Hello")

คำสั่ง print ใช้แสดงผลข้อความทางหน้าจอ

3. Scratch Block
when clicked
say Hello for 2 seconds
03

การจัดการข้อมูลสารสนเทศ

Data & Information

วงจรการประมวลผลข้อมูล (IPOS Cycle)

1. Input

รับข้อมูลเข้า
(Data)

2. Process

ประมวลผล
(Calculate/Sort)

3. Output

แสดงผลลัพธ์
(Information)

4. Storage (จัดเก็บ)

ข้อมูล vs สารสนเทศ

DATA

ข้อเท็จจริงดิบๆ ที่ยังไม่ผ่านการประมวลผล เช่น ตัวเลข 45, 50, 48 (คะแนนสอบรายคน)

INFORMATION

ข้อมูลที่ผ่านการประมวลผลแล้ว มีความหมาย เช่น คะแนนเฉลี่ย = 47.6 (นำไปตัดเกรดได้)

ลักษณะสารสนเทศที่ดี

  • ความถูกต้อง (Accuracy): ไม่มีข้อผิดพลาด
  • ความทันสมัย (Timeliness): เป็นปัจจุบัน
  • ความสมบูรณ์ (Completeness): ครบถ้วนทุกด้าน
  • ตรวจสอบได้ (Verifiability): มีแหล่งที่มาชัดเจน
04

ความปลอดภัย & จริยธรรม

Safety & Ethics

รู้จัก Malware (มัลแวร์)

Virus (ไวรัส)

โปรแกรมที่แฝงตัวมากับไฟล์ แตกตัวทำลายข้อมูล (ต้องมีพาหะ)

Worm (หนอน)

แพร่กระจายเองผ่านระบบเครือข่าย/เน็ต ทำให้เน็ตช้า (ไม่ต้องมีพาหะ)

Trojan Horse (ม้าโทรจัน)

ปลอมตัวเป็นโปรแกรมดีๆ เพื่อหลอกให้เราติดตั้ง แล้วขโมยข้อมูล

3 กฎเหล็กความปลอดภัย

  • 1
    ตั้งรหัสผ่านให้ยาก: อย่างน้อย 8 ตัว มีตัวเลข+อักษร+สัญลักษณ์
  • 2
    Think Before Share: คิดก่อนแชร์หรือโพสต์ เพราะ Digital Footprint อยู่ตลอดไป
  • 3
    Log out เสมอ: เมื่อใช้เครื่องสาธารณะ เพื่อกันคนสวมรอย

จริยธรรม (Ethics)

"ใจเขาใจเรา" ไม่ใช้คอมพิวเตอร์ทำร้ายหรือรบกวนผู้อื่น เคารพความเป็นส่วนตัวและลิขสิทธิ์
