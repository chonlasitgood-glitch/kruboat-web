01

การจัดการข้อมูล

Data Science Fundamentals

ทฤษฎี: ลำดับชั้นของข้อมูล (DIKW Pyramid)

  • Data (ข้อมูล): ข้อเท็จจริงดิบๆ เช่น ตัวเลข 25, 30 (ยังไม่รู้ว่าคืออะไร)
  • Information (สารสนเทศ): ข้อมูลที่ผ่านการประมวลผล เช่น อุณหภูมิเฉลี่ย 25 องศา
  • Knowledge (ความรู้): การเข้าใจรูปแบบ เช่น อุณหภูมิ 25 องศาคืออากาศเย็นสบาย
  • Wisdom (ภูมิปัญญา): การนำไปตัดสินใจ เช่น วันนี้อากาศดี ควรจัดกิจกรรมกลางแจ้ง

1. การเก็บรวบรวมข้อมูล (Data Collection)

แบ่งประเภทข้อมูลตามคุณลักษณะ (Attributes) และแหล่งที่มา:

แบ่งตามแหล่งที่มา
  • ปฐมภูมิ (Primary): เก็บเอง (สัมภาษณ์, แบบสอบถาม, เซนเซอร์ IoT)
  • ทุติยภูมิ (Secondary): คนอื่นเก็บไว้ (สถิติรัฐ, รายงานวิจัย, API)
แบ่งตามรูปแบบ
  • เชิงปริมาณ (Quantitative): ตัวเลขคำนวณได้ (น้ำหนัก, ส่วนสูง)
  • เชิงคุณภาพ (Qualitative): ความรู้สึก, ความเห็น, สีที่ชอบ

2. การประมวลผลข้อมูล (Data Processing)

Data Cleaning (การทำความสะอาดข้อมูล)

ก่อนนำข้อมูลไปใช้ ต้องจัดการสิ่งผิดปกติ:

  • Missing Values: ค่าว่าง (อาจเติมด้วยค่าเฉลี่ย หรือตัดทิ้ง)
  • Outliers: ค่าที่โดดผิดปกติ (เช่น อายุ 200 ปี)
  • Inconsistency: ความไม่สม่ำเสมอ (เช่น ชาย, Male, M)

Data Analysis Types

Descriptive: เกิดอะไรขึ้น? (สรุปผลอดีต)
Predictive: จะเกิดอะไรขึ้น? (พยากรณ์อนาคต)

Big Data (5Vs)

ข้อมูลขนาดใหญ่ต้องมีลักษณะ 5 อย่าง:

  • Volume: ปริมาณมหาศาล
  • Velocity: เปลี่ยนแปลงรวดเร็ว
  • Variety: หลากหลายรูปแบบ (ภาพ, เสียง, ข้อความ)
  • Veracity: ความถูกต้องน่าเชื่อถือ
  • Value: มีคุณค่าทางธุรกิจ/สังคม

เครื่องมือ (Tools)

  • Spreadsheet
    Excel / Google Sheets (งานพื้นฐาน)
  • Python (Pandas)
    Data Science / Automation
  • Data Studio / PowerBI
    Dashboard Visualization
02

ความน่าเชื่อถือ

Information Literacy

หลักการประเมิน PROMPT

วิธีตรวจสอบข้อมูลก่อนเชื่อหรือแชร์:

P
Presentation: การนำเสนอเป็นมืออาชีพหรือไม่?
R
Relevance: ตรงกับหัวข้อที่หาหรือไม่?
O
Objectivity: มีอคติหรือเจตนาแอบแฝงไหม?
M
Method: วิธีการเก็บข้อมูลน่าเชื่อถือไหม?
P
Provenance: ระบุแหล่งที่มาชัดเจนไหม?
T
Timeliness: ข้อมูลเป็นปัจจุบันหรือไม่?

เทคนิคการสืบค้น (Search Operators)

"..." Exact Match

ค้นหาคำตรงตัวเป๊ะๆ เช่น "โลกร้อน"

AND / OR Logic

แมว AND หมา (ต้องมีทั้งคู่), แมว OR หมา (มีอย่างใดอย่างหนึ่ง)

- (ลบ) Exclude

ตัดคำที่ไม่ต้องการ เช่น รถยนต์ -มือสอง

site: Domain

ระบุเว็บ เช่น site:ac.th (หาเฉพาะเว็บสถานศึกษา)

ภัยเงียบโลกออนไลน์

  • Cyberbullying: การกลั่นแกล้ง รังแก หรือสร้างความเกลียดชังออนไลน์
  • Digital Footprint: ร่องรอยดิจิทัลที่ลบไม่ออก (โพสต์, ประวัติการค้นหา)
  • Phishing: เว็บ/อีเมลปลอมหลอกให้กรอกรหัสผ่าน

การรู้เท่าทันสื่อ (Media Literacy)

  • Deepfake: AI ตัดต่อใบหน้า/เสียง ต้องสังเกตการกระพริบตาหรือเงา
  • Clickbait: พาดหัวเวอร์วัง ล่อให้กดเข้าไปดูโฆษณา
  • Echo Chamber: เห็นแต่ข้อมูลที่ตัวเองชอบ เพราะ AI เลือกให้

กฎหมาย & ลิขสิทธิ์

Computer Law & PDPA

พ.ร.บ. คอมพิวเตอร์ฯ (ฉบับที่ 2)

  • มาตรา 14 (1) - ข้อมูลเท็จ/บิดเบือน

    นำเข้าข้อมูลอันเป็นเท็จ สร้างความเสียหาย/ตื่นตระหนก (โทษ: คุก 5 ปี / ปรับ 1 แสน)

  • มาตรา 16 - ตัดต่อภาพ

    ตัดต่อภาพผู้อื่นให้เสียชื่อเสียง อับอาย (โทษ: คุก 3 ปี / ปรับ 2 แสน)

  • มาตรา 5-7 - การเข้าถึงระบบ

    Hack เข้าระบบ/ข้อมูลผู้อื่นโดยมิชอบ (แม้ไม่ได้ขโมยอะไรก็ผิด)

PDPA (พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล)

กฎหมายที่คุ้มครองสิทธิในข้อมูลส่วนตัวของเรา

  • ต้องขอความยินยอม (Consent) ก่อนเก็บข้อมูล
  • ใช้ข้อมูลตามวัตถุประสงค์ที่แจ้งเท่านั้น
  • เจ้าของข้อมูลมีสิทธิขอให้ลบหรือระงับการใช้ข้อมูล

แอปพลิเคชัน & IoT

Application Development

สถาปัตยกรรม IoT (IoT Architecture)

1. Perception Layer

ชั้นรับรู้: เซนเซอร์ (วัดอุณหภูมิ, จับการเคลื่อนไหว), RFID, QR Code

2. Network Layer

ชั้นเครือข่าย: ส่งข้อมูลผ่าน WiFi, Bluetooth, 5G, NB-IoT ไปยัง Cloud

3. Processing Layer

ชั้นประมวลผล: Cloud Computing วิเคราะห์ข้อมูล Big Data

4. Application Layer

ชั้นใช้งาน: แอปพลิเคชันบนมือถือ, Smart Home Dashboard

ตัวอย่างการประยุกต์ใช้: Smart Farm

Input: เซนเซอร์วัดความชื้นในดิน และเซนเซอร์วัดแสงแดด ส่งค่าไปยังบอร์ด KidBright

Process: บอร์ดตรวจสอบเงื่อนไข "ถ้าความชื้น < 40% ให้เปิดน้ำ"

Output: สั่งงาน Relay ให้เปิดปั๊มน้ำ และแจ้งเตือนเกษตรกรผ่าน LINE Notify

if (humidity < 40) {
  pump.on();
  notifyLine("รดน้ำต้นไม้แล้วครับ");
} else {
  pump.off();
}
